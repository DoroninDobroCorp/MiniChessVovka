# Changelog: Режим самообучения

## Дата: 14 октября 2024

### 🎯 Цель
Создать режим самообучения, где AI играет сам с собой и в 20% случаев выбирает второй лучший ход для исследования альтернативных веток.

### ✨ Новые функции

#### 1. Модификации в `ai.py`
- **`find_best_move()`** теперь принимает параметр `return_top_n`:
  - `return_top_n=1` (по умолчанию) - возвращает лучший ход (обратная совместимость)
  - `return_top_n>1` - возвращает список `[(move, score), ...]` топ-N ходов
  
- **`minimax()`** теперь принимает параметр `return_all_scores`:
  - `return_all_scores=False` (по умолчанию) - возвращает `(best_move, best_score)`
  - `return_all_scores=True` - возвращает `(best_move, best_score, all_results)`

#### 2. Новый файл `self_play.py`
Полнофункциональный скрипт самообучения с:
- AI vs AI режимом
- Исследованием альтернатив (20% - второй лучший ход)
- Graceful shutdown (безопасная остановка по Ctrl+C)
- Автоматическим сохранением кэша после каждого хода
- Подробной статистикой и мониторингом
- Защитой от выбора неоптимальных ходов при мате

#### 3. Новый файл `train.sh`

- Информационным баннером
- Простым использованием: `./train.sh`

### 🔧 Технические детали

#### Алгоритм выбора хода:
```python
1. Получить топ-2 лучших хода с оценками
2. Если только один ход → выбрать его
3. Если лучший ход - мат → выбрать его (не исследуем)
4. С вероятностью 20% → выбрать 2-й лучший
5. С вероятностью 80% → выбрать лучший
```

#### Сохранение прогресса:
- Кэш сохраняется в `move_cache.db` после каждого хода
- При прерывании (Ctrl+C) кэш сохраняется перед выходом
- Используется SQLite для надежного хранения

#### Graceful shutdown:
- Обработчики сигналов: `SIGINT` и `SIGTERM`
- Флаг `shutdown_requested` проверяется в главном цикле
- Корректное завершение текущего хода перед выходом
- Сохранение финальной статистики

### 🛡️ Безопасность и стабильность

1. **Обратная совместимость**:
   - Существующий код не затронут
   - `find_best_move()` работает как раньше при вызове без параметров
   - Основная игра (`main.py`, `gui.py`) работает без изменений

2. **Graceful shutdown**:
   - Безопасная остановка по Ctrl+C
   - Гарантированное сохранение кэша
   - Вывод финальной статистики

3. **Защита от ошибок**:
   - Проверка на единственный легальный ход
   - Приоритет мата (не исследуем при мате)
   - Обработка исключений при сохранении кэша
   - Лимит ходов на партию (200) для предотвращения зацикливания

### 🚀 Использование

**Запуск:**
```bash
./train.sh
```

**Остановка:**
```bash
Ctrl+C (один раз, подождите завершения хода)
```

**Настройка (в `self_play.py`):**
```python
depth = 6              # 4-8 оптимально
exploration_rate = 0.2 # 0.1-0.3 рекомендуется
num_games = None       # None = бесконечно
```

### 📈 Ожидаемые результаты

- **Глубина 6**: ~5-15 минут на ход (зависит от процессора)
- **Размер кэша**: ~50-100 позиций на партию
- **Соотношение побед**: Должно быть примерно равномерным (50/50)
- **Качество игры**: Улучшается с ростом кэша

### ⚠️ Известные ограничения

1. **Производительность**: На глубине 6+ ходы занимают много времени
2. **Размер БД**: Может вырасти до нескольких МБ после многих игр
3. **Память**: При больших глубинах может требоваться много RAM
4. **Параллелизм**: Пока только одна партия за раз
