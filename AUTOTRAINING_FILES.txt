==============================================
Файлы системы автоматического обучения
==============================================

ОСНОВНЫЕ СКРИПТЫ
================

1. autorun_training.sh
   - Главный скрипт с автоперезапуском при сбоях
   - Мониторит здоровье процесса
   - Отправляет graceful SIGTERM при остановке
   - До 5 попыток перезапуска
   ✅ УСТАНОВЛЕН И РАБОТАЕТ

2. scheduled_self_play.py
   - Логика самообучения AI vs AI
   - Depth 6, 20% исследование
   - Детальное логирование каждого хода
   - Сохранение в кэш БД
   ✅ РАБОТАЕТ

3. run_scheduled_training.sh
   - Управление расписанием (старый способ)
   - Может использоваться для ручного запуска
   ✅ СОХРАНЕН, НО ТЕПЕРЬ SYSTEMD ВМЕСТО ЭТОГО

МОНИТОРИНГ И УПРАВЛЕНИЕ
========================

4. check_training_health.sh
   - Быстрая проверка здоровья процесса
   - Проверяет PID, CPU, MEM, health file
   - Вывод последних логов
   Команда: ./check_training_health.sh
   ✅ ГОТОВ

5. training_dashboard.sh
   - Live dashboard с обновлением каждые 10 сек
   - Показывает статус, использование ресурсов, логи
   Команда: ./training_dashboard.sh
   ✅ ГОТОВ

SYSTEMD (система автозапуска)
==============================

6. minichesstrain.service
   - systemd service для управления обучением
   - Запускает autorun_training.sh
   - Автоперезапуск при ошибке (RestartSec=10)
   - Ограничения: 80% CPU, 4GB RAM
   Установлен в: /etc/systemd/system/
   ✅ УСТАНОВЛЕН И АКТИВИРОВАН

7. minichesstrain.timer
   - systemd timer для расписания
   - Проверяет каждые 30 минут
   - Запускает service если нужно
   - Действует на расписание (0-8 UTC)
   Установлен в: /etc/systemd/system/
   ✅ УСТАНОВЛЕН И АКТИВИРОВАН

УСТАНОВЩИК
===========

8. setup_autotraining.sh
   - Автоматический установщик systemd
   - Копирует файлы, перезагружает systemd
   - Активирует service и timer
   Команда: sudo bash setup_autotraining.sh
   ✅ УЖЕ ВЫПОЛНЕН

ЛОГИ И ПРОГРЕСС
================

9. training_log.txt
   - Подробный лог каждого хода
   - Каждый ход и партия логируется
   - Растет во время обучения
   Просмотр: tail -f training_log.txt

10. monitor.log
    - Логи мониторинга и перезапусков
    - Информация о попытках запуска
    - Сигналы и graceful shutdown

11. training_progress.txt
    - Файл прогресса (количество игр, статистика)
    - Обновляется после каждой партии
    - Содержит статистику

12. scheduler.log
    - Логи расписания (старая система)
    - Сохранен для истории

ДОКУМЕНТАЦИЯ
=============

13. AUTOTRAINING_SETUP.md
    - Полная документация по системе
    - Инструкции по установке и использованию
    - Отладка и конфигурация

14. SETUP_COMPLETE.md
    - Резюме установки
    - Текущий статус
    - Быстрые команды

15. AUTOTRAINING_FILES.txt
    - Этот файл

==============================================
БЫСТРЫЕ КОМАНДЫ
==============================================

Проверка здоровья:
  ./check_training_health.sh

Live мониторинг:
  ./training_dashboard.sh

Статус service:
  sudo systemctl status minichesstrain.service

Просмотр логов (real-time):
  sudo journalctl -u minichesstrain.service -f

Запустить обучение:
  sudo systemctl start minichesstrain.service

Остановить:
  sudo systemctl stop minichesstrain.service

Перезапустить:
  sudo systemctl restart minichesstrain.service

==============================================
РАСПИСАНИЕ
==============================================

Время работы: 00:00 - 08:00 UTC (ночные часы)

Проверка расписания: каждые 30 минут
Запуск: в 00:00 UTC (если timer активен)
Остановка: в 08:00 UTC

Текущее время: используй `date` или `timedatectl`

==============================================
АРХИТЕКТУРА
==============================================

systemd timer (каждые 30 мин)
         ↓
проверка расписания (0-8 UTC?)
         ↓
если ДА → systemd service
         ↓
autorun_training.sh
         ├─ Попытка запуска (#1)
         ├─ Мониторинг здоровья
         ├─ Обнаружение краша
         └─ Перезапуск (#2-5)
         ↓
scheduled_self_play.py
         ├─ Загрузка кэша из БД
         ├─ Партия 1: AI vs AI depth 6
         ├─ Партия 2, 3, ...
         ├─ Сохранение результатов
         └─ Обновление progress.txt
         ↓
Логирование:
  ├─ training_log.txt (file)
  ├─ monitor.log (file)
  └─ journalctl (systemd)
