# Автоматическое обучение Mini Chess AI

## Описание

Система автоматического обучения, которая запускает самообучение AI каждый день с 2:00 до 10:00, используя только 7 из 8 процессорных ядер, чтобы не перегружать систему.

## Возможности

- ✅ **Автоматический запуск**: Обучение запускается каждый день в 2:00 и останавливается в 10:00
- ✅ **Ограничение CPU**: Использует только 7 ядер из 8, оставляя одно ядро свободным
- ✅ **Подробное логирование**: Все ходы записываются с временными метками
- ✅ **Отслеживание прогресса**: Специальный файл с текущим прогрессом обучения
- ✅ **Graceful shutdown**: Безопасная остановка с сохранением всего прогресса
- ✅ **Мониторинг**: Легко проверить статус и прогресс обучения

## Быстрый старт

### Установка

```bash
cd /srv/MiniChessVovka
chmod +x setup_auto_training.sh
./setup_auto_training.sh
```

Скрипт установки:
1. Установит права выполнения на все необходимые файлы
2. Проверит наличие утилиты `taskset` для ограничения CPU
3. Настроит задачу cron для автоматического запуска
4. Создаст файлы логов

### Проверка работы

После установки вы можете:

```bash
# Проверить, что cron задача добавлена
crontab -l

# Посмотреть текущий статус
./run_scheduled_training.sh status

# Посмотреть прогресс обучения
cat training_progress.txt

# Следить за логом в реальном времени
tail -f training_log.txt
```

## Управление

### Ручной запуск (вне расписания)

```bash
./run_scheduled_training.sh start
```

### Остановка обучения

```bash
./run_scheduled_training.sh stop
```

### Проверка статуса

```bash
./run_scheduled_training.sh status
```

Выведет:
- PID процесса обучения
- Использование CPU и памяти
- Последние 10 строк лога

## Файлы и логи

### training_log.txt
Подробный лог всех ходов и событий обучения. Каждая строка содержит:
- Временную метку
- Номер партии и хода
- Выбранный ход с оценкой
- Время расчета хода
- Результаты партий

Пример:
```
[2024-10-18 03:15:42] --- Ход 15 (Белые) ---
[2024-10-18 03:15:50]   Лучший ход (стандартный выбор): d3-d4, оценка: 245.0
[2024-10-18 03:15:50]     2-й вариант: c2-c3, оценка: 230.0
[2024-10-18 03:15:50]     Время расчета: 8.3с
```

### training_progress.txt
Сводка текущего прогресса обучения. Обновляется после каждой партии:
- Время работы
- Количество сыгранных игр
- Размер кэша (количество изученных позиций)
- Статистика результатов (маты, паты)
- Средние показатели

### scheduler.log
Лог работы планировщика:
- Когда проверялось расписание
- Когда запускалось/останавливалось обучение
- Ошибки планировщика (если есть)

### move_cache.db
База данных SQLite с кэшем всех изученных позиций. Растет с каждой партией.

## Настройка параметров

### Изменение времени работы

Отредактируйте файл `run_scheduled_training.sh`:

```bash
# Временное окно (часы)
START_HOUR=2   # Начало обучения (2:00)
END_HOUR=10    # Конец обучения (10:00)
```

### Изменение количества используемых ядер

Отредактируйте файл `run_scheduled_training.sh`:

```bash
# CPU ограничения
CPU_MASK="0-6"  # Использовать ядра 0-6 (7 ядер)
# Для использования 6 ядер: CPU_MASK="0-5"
# Для использования 4 ядер: CPU_MASK="0-3"
```

### Изменение параметров AI

Отредактируйте файл `scheduled_self_play.py`, функцию `main()`:

```python
def main():
    num_games = None        # None = бесконечно
    depth = 6              # Глубина поиска (4-8)
    exploration_rate = 0.2 # 20% исследования
```

**Рекомендации по глубине:**
- `depth = 4`: Быстро (~1-3 мин на ход)
- `depth = 6`: Оптимально (~5-15 мин на ход) ⭐
- `depth = 8`: Очень сильно (~20-60 мин на ход)

### Изменение частоты проверки

По умолчанию планировщик проверяет расписание каждые 15 минут. Для изменения:

```bash
# Редактируем cron
crontab -e

# Измените */15 на желаемое значение:
*/10 * * * * /srv/MiniChessVovka/run_scheduled_training.sh  # Каждые 10 минут
*/5 * * * * /srv/MiniChessVovka/run_scheduled_training.sh   # Каждые 5 минут
```

## Расчет времени завершения обучения

Время полного обучения зависит от:
1. Глубины поиска
2. Количества партий
3. Производительности процессора

**Примерные расчеты:**

При `depth = 6` на современном процессоре:
- Одна партия: ~30-60 ходов
- Время на ход: ~5-10 минут
- Время партии: ~3-6 часов
- За ночь (8 часов): ~1-2 партии

**Для 100 партий:**
- При 1 партии/ночь: ~100 дней (3 месяца)
- При 2 партиях/ночь: ~50 дней (1.5 месяца)

**Ускорение обучения:**
1. Уменьшить глубину до 4-5
2. Использовать все 8 ядер вместо 7
3. Увеличить время работы (например, с 1:00 до 11:00)

## Troubleshooting

### Обучение не запускается автоматически

1. Проверьте, что задача cron установлена:
   ```bash
   crontab -l | grep run_scheduled_training
   ```

2. Проверьте права выполнения:
   ```bash
   ls -la run_scheduled_training.sh scheduled_self_play.py
   ```

3. Проверьте лог планировщика:
   ```bash
   tail -f scheduler.log
   ```

### Процесс использует все ядра

1. Проверьте, установлен ли `taskset`:
   ```bash
   which taskset
   ```

2. Если нет, установите:
   ```bash
   sudo apt-get install util-linux
   ```

3. Проверьте, что процесс запущен с `taskset`:
   ```bash
   ps aux | grep scheduled_self_play
   ```

### Обучение останавливается раньше времени

1. Проверьте логи на ошибки:
   ```bash
   tail -100 training_log.txt
   grep -i error training_log.txt
   ```

2. Проверьте, хватает ли памяти:
   ```bash
   free -h
   ```

3. Проверьте логи системы:
   ```bash
   journalctl -xe | grep python
   ```

### База данных слишком большая

```bash
# Проверить размер
ls -lh move_cache.db

# Очистить кэш (начать обучение заново)
rm move_cache.db

# Или сделать резервную копию
mv move_cache.db move_cache_backup.db
```

## Советы по использованию

1. **Первый запуск**: Запустите вручную днем, чтобы убедиться, что все работает:
   ```bash
   ./run_scheduled_training.sh start
   ```

5. **Планирование**: Учитывайте, что первые партии самые быстрые (меньше позиций в кэше)

## Технические детали

### Архитектура

```
setup_auto_training.sh          # Скрипт установки
    ↓
cron (каждые 15 минут)
    ↓
run_scheduled_training.sh       # Планировщик
    ↓ (если 2:00-10:00)
taskset -c 0-6                  # Ограничение CPU
    ↓
scheduled_self_play.py          # Обучение
    ↓
training_log.txt                # Подробный лог
training_progress.txt           # Сводка
move_cache.db                   # База знаний
```

### Безопасность

- Graceful shutdown: процесс корректно обрабатывает SIGTERM
- Автосохранение: кэш сохраняется после каждого хода
- PID файл: предотвращает запуск дубликатов
- Timeout: принудительная остановка через 30 секунд если процесс завис

### Производительность

Использование ресурсов:
- **CPU**: 7 ядер на 100% (одно ядро свободно)
- **RAM**: ~100-500 MB (зависит от размера кэша)
- **Disk**: растущая база данных (~10-100 MB)
- **I/O**: минимальный (только запись логов и БД)
