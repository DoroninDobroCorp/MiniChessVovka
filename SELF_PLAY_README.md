# Режим самообучения Mini Chess AI

## Описание

Режим самообучения позволяет AI играть сам с собой и автоматически накапливать опыт в кэше ходов. В 20% случаев AI выбирает второй лучший ход вместо самого лучшего, чтобы исследовать альтернативные ветки игрового дерева и избежать зацикливания на одних и тех же вариантах.

## Основные возможности

- ✅ **AI против себя**: Оба игрока управляются AI
- ✅ **Исследование альтернатив**: 20% ходов - второй лучший вместо лучшего
- ✅ **Graceful shutdown**: Безопасная остановка с сохранением прогресса при Ctrl+C
- ✅ **Автоматическое сохранение**: Кэш сохраняется в БД после каждого хода
- ✅ **Подробная статистика**: Отслеживание результатов и прогресса

## Параметры настройки

Вы можете изменить параметры в `self_play.py` в функции `main()`:

```python
def main():
    num_games = None        # None = бесконечно, или укажите число (например, 100)
    depth = 6              # Глубина поиска (4-8 оптимально)
    exploration_rate = 0.2 # Вероятность выбора 2-го хода (0.0 - 1.0)
    
    run_self_play_training(num_games, depth, exploration_rate)
```

### Рекомендации по параметрам:

- **depth = 4**: Быстрое обучение (~1-3 минуты на ход)
- **depth = 6**: Оптимальный баланс (~5-15 минут на ход) ⭐
- **depth = 8**: Очень сильная игра (~20-60 минут на ход)

- **exploration_rate = 0.2**: Рекомендуемое значение (20%)
- **exploration_rate = 0.3**: Больше исследования, медленнее сходится
- **exploration_rate = 0.1**: Меньше исследования, может застрять в локальном максимуме

## Как это работает

1. **Запуск партии**: AI создает новую позицию и начинает игру
2. **Выбор хода**: 
   - Получает топ-2 лучших хода с оценками
   - С вероятностью 20% выбирает 2-й лучший ход
   - С вероятностью 80% выбирает лучший ход
   - При мате всегда выбирает матующий ход
3. **Сохранение**: После каждого хода кэш сохраняется в `move_cache.db`
4. **Статистика**: В конце каждой партии выводится подробная статистика

## Остановка и возобновление

### Остановка
Нажмите `Ctrl+C` один раз. Скрипт:
- Корректно завершит текущую партию
- Сохранит весь кэш в базу данных
- Выведет финальную статистику

⚠️ **Важно**: Не убивайте процесс принудительно (kill -9), иначе последние изменения могут не сохраниться!

### Возобновление
Просто запустите скрипт снова:
```bash
./train.sh
```
Кэш автоматически загрузится из базы данных и обучение продолжится с того же уровня.

## Мониторинг прогресса

Во время работы выводится:
- Номер текущей партии
- Текущий ход и игрок
- Выбранный ход и его оценка
- Информация об исследовании (если выбран 2-й лучший ход)
- Время расчета каждого хода
- Статистика после каждой партии

Пример вывода:
```
--- Ход 15 (Белые) ---
  ИССЛЕДОВАНИЕ: Выбираем 2-й лучший ход
    1-й: d3-d4, оценка: 245.0
    2-й: c2-c3, оценка: 230.0 ← ВЫБРАН
  Время расчета: 8.3с
```

## Статистика

После каждой партии выводится:
- Результат (мат/пат/лимит ходов)
- Победитель (если есть)
- Количество ходов
- Длительность партии
- Среднее время хода

Общая статистика включает:
- Всего игр
- Процент матов / патов
- Соотношение побед белых/черных
- Среднее количество ходов
- Среднее время партии
- Размер кэша (количество сохраненных позиций)

## База данных

Кэш ходов хранится в файле `move_cache.db` (SQLite).

### Просмотр размера кэша
```bash
sqlite3 move_cache.db "SELECT COUNT(*) FROM move_cache;"
```

### Очистка кэша (начать обучение с нуля)
```bash
rm move_cache.db
```

## Советы по использованию

1. **Запускайте на ночь**: Процесс долгий, удобно запускать когда компьютер не используется
2. **Начните с меньшей глубины**: Если 10+ минут на ход - слишком долго, уменьшите depth до 4-5
3. **Мониторьте размер кэша**: Он растет с каждой партией и улучшает силу игры
4. **Не прерывайте часто**: Дайте сыграть хотя бы 5-10 партий для накопления опыта
5. **Периодически играйте против AI**: Проверяйте, стал ли он сильнее

## Технические детали

### Почему 20%?
- Это баланс между эксплуатацией (использование лучших ходов) и исследованием (поиск новых стратегий)
- Слишком большое исследование → AI не учится на своих ошибках
- Слишком маленькое → AI застревает в одних и тех же вариантах

### Когда AI не исследует?
- Если найден мат (всегда играет матующий ход)
- Если есть только один легальный ход

### Размер кэша
- Каждая позиция хранится с привязкой к глубине поиска
- При глубине 6 одна партия добавляет ~50-100 позиций
- База данных может вырасти до нескольких МБ после многих игр

## Troubleshooting

### "Ход занимает очень долго"
→ Уменьшите `depth` до 4 или 5

### "Кэш не сохраняется"
→ Проверьте права на запись в директорию проекта

### "Ошибка импорта"
→ Убедитесь, что все файлы проекта (ai.py, gamestate.py и т.д.) в той же директории

### "Process killed"
→ Возможно, не хватает памяти. Закройте другие программы или уменьшите глубину
